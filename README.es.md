# Frontend Mentor - Fylo Dark Theme Landing Page Solution

Esta es mi solución al desafío **Fylo Dark Theme Landing Page** de Frontend Mentor. Este proyecto se centra en la creación de una landing page completamente responsive con un tema oscuro, una cuadrícula de funcionalidades, testimonios y un formulario call to action, siguiendo buenas prácticas modernas de desarrollo frontend con HTML, CSS y JavaScript.

El desafío fue una gran oportunidad para practicar HTML semántico, layouts responsive con Flexbox, propiedades personalizadas de CSS y validación de formularios en tiempo real utilizando JavaScript puro.

## Visión general
Este proyecto es una landing page responsive para Fylo, diseñada con un tema oscuro y optimizada para dispositivos móviles, tabletas y escritorio. El diseño se adapta de forma fluida a diferentes tamaños de pantalla, ofreciendo una experiencia de usuario limpia y atractiva.

La página incluye una sección hero, una cuadrícula de funcionalidades, una sección de productividad, testimonios, un formulario de registro con validación y un footer detallado con navegación y enlaces a redes sociales. El objetivo fue replicar lo más fielmente posible el diseño proporcionado, manteniendo el código limpio, semántico y fácil de mantener.

---

## El desafío
Los usuarios deben poder:

- Ver el diseño óptimo según el tamaño de pantalla de su dispositivo.
- Experimentar un diseño responsive con enfoque **mobile-first**.
- Navegar de forma fluida entre secciones mediante enlaces ancla.
- Ver estados hover y focus en todos los elementos interactivos.
- Introducir una dirección de correo electrónico y recibir validación instantánea.
- Disfrutar de un diseño que se ajuste fielmente al original de Frontend Mentor.

## Mi proceso
- Estructuré la página utilizando **elementos semánticos de HTML5** como `nav`, `main`, `section`, `article`, `form` y `footer`.
- Seguí un enfoque **mobile-first**, mejorando progresivamente el diseño mediante media queries.
- Construí el layout utilizando **Flexbox** para la alineación y distribución de los componentes.
- Utilicé **propiedades personalizadas de CSS (variables)** para gestionar los colores y mantener un tema oscuro consistente.
- Apliqué la **metodología BEM** para el nombrado de clases, manteniendo el CSS modular y escalable.
- Estilicé los elementos interactivos con estados hover y focus para mejorar la usabilidad y la accesibilidad.
- Implementé **validación de email en tiempo real** usando JavaScript puro y expresiones regulares.
- Controlé los estados de la interfaz alternando clases CSS desde JavaScript.
- Aseguré una clara separación de responsabilidades entre HTML, CSS y JavaScript.
